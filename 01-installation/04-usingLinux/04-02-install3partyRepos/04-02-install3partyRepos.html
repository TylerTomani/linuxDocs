 The standard enterprise linux repositories contain alot of
server packages, but not alot of GUI programs. To add desktop GUIs,
you need to add third-party software repositories.

 There are several ways of doing this

 The first way shown here adds the repository automatically.

 The second way, the repository will reside in a package that we will
install from the standara enterprise linux repo

 The last repo, we will download a RPM package from the internet
and manually install it

1.

 First, download chrome in firefox

2.

 This should recognize that you are on linux,
Notice the will add the Google repository, this makes
future updates secure and easy.

 Make sure 64 bit .rpm (For fedora.. is selected

3.

 When prompted save the package. By default enterprise linux will save it in the downloads directory

4.

 Install using the command line, type,

cd ~/Downloads sudo dnf install google-chrome*

 use the * because it's a wildcard and will install any file
that starts with google-chrome

5.

 Once install is complete, check if it has been added to the software repository.
type,

sudo dnf repolist

 Downloading software this way is always better
It comes with sometype of hash symbol, that dnf looks for. If software doesn't have (often from nefarious sites), dnf will not install it. This makes it harder for hackers and malware

6. Dark Table

 This packages too resides in a third-party repo. It is a photo editor,
like gimp, but works well with raw photos.

 Dark table is in the EPEL repo, type,

sudo dnf install -y epel-release

 This software is essential for installing common GUI programs
